Siemens Healthcare

Software tester - Intern
1st July 2016 - 31st October 2016

About the Internship

Being a part of Siemens was indeed a great opportunity and my best career experience so far. Joining such a corporate size company was difficult at first, considering that It was my first experience in the field outside class room and freelancing. I have joined one of the SCRUM teams, which worked on the Central Image Processing (CIP) module of application called Syngo.via. Since Syngo via was running on .NET framework, I had a change to apply skills, which I have acquared from my previous projects.

Responsibilities and Tasks

One of the pros of my internship was that my tasks changed based on my knowledge and experience with the application. I began with writing Unit tests using Moq library. My task was to improve test coverage, preferably of classes with higher defect occurences. I have quickly learned about the importance of testable architecture and design, because I was not able to properly test some part of the poorly written / legacy code.

Later I started to write automated tests. I was focusing on this during the majority of the internship, since automated UI tests were crutial for such front-end heavy application. I was given the manual test specification as well as the requirement specification and my task was to write tests for features that have not yet been tested. The automation took advantage of the command pattern and set of test steps. The key point was to write the tests fluently, so that they resemble the manual test specification as much as possible. I have applied the knowledge of automation in my later projects - Spots and Georgia tech library.

I was also part of manual testing that the whole team participated upon after every small release. The team would slip the manual test specification and collectevely test the part of the system its responsible for. If any defects were found, they were documented and later verified by the product owner.

Finally, I was also working on improvment of the automated test step library. Some features to be tested required test steps that were not supported, so I was to implement them.

Spots

Position: Owner
Period: 1st October 2016 - 31st Jun 2017

What is Spots?

Spots in a nutshell is a centrilized booking system. It is designed to help users to manage their meetings, events and their time overall. A large part of the system also focuses on communication - users are able to interact, create meetings together as well as to see who is available and when. The application helps users to find a perfect spot for their meetings - whether it is a table at the restaurant or a classroom for university team meeting. Spots started as a university project and a company was later on created as a part of my internship at UCN's next step.

Team

Having a company gave me an unique experience of working with people. I had the priviledge to have an amazing team, where every one contributed differently - from FE and BE development, to project management, marketing and design.

Technologies behind spots

Spots was developed as a web application, using ASP.Net MVC and C#. When it comes to databases, we chose different databases for different purposes - SQL server was used for authentication and NoSQL database mongoDB was used to handle information about the users, spots, locations and events as well as exception and activity logs.
On the client side, Knockout JS and MVVM pattern was used, together with bootstrap and jquery.

Technical outcome

From the technological point of view, I have learned a lot about MVC architecture and I strenghtened my knowledge in C#, Object oriented programming, Design pattern and generally improved the quality of my code. The biggest value of the project is that it had much longer lifespan than other university projects I have done. This taught me the importance of writing clean, testable code according to the SOLID principles.
